---
name: skill-registry
description: >
  Generate or update the project conventions file. Scans skills and project conventions.
  Outputs to openspec/conventions.md (canonical) + optional .github/instructions/ copy for Copilot.
  Trigger: "update skills", "skill registry", "actualizar skills", or after installing/removing skills.
---

## Steps

### 1. Scan Skills
Glob for `*/SKILL.md` across:
- **User-level**: `~/.claude/skills/`, `~/.copilot/skills/`
- **Project-level**: `{root}/.claude/skills/`, `{root}/.github/skills/`, `{root}/skills/`

**SKIP**: `sdd-*`, `_shared`, `skill-registry`. Deduplicate (project-level wins over user-level).

For each skill: read frontmatter + rules section (skip prose). Extract name, trigger, and generate compact rules (5–15 lines: actionable constraints, patterns, gotchas only — no purpose statements).

### 2. Scan Project Conventions
Check for: `CLAUDE.md` (project root or `.claude/`), `.cursorrules`, `.github/copilot-instructions.md`.
Detect stack from `openspec/config.yaml` if exists.

### 3. Write `openspec/conventions.md`
```markdown
# Project Conventions
_Auto-generated by skill-registry. Run /skill-registry to update._

## Stack
{from openspec/config.yaml context field, or "Not yet initialized — run /sdd-init"}

## Skills Available
| Trigger | Skill | Description |
|---------|-------|-------------|
{one row per skill}

## Compact Rules
### {skill-name}
{5-15 actionable rules}

## Project Conventions
| File | Path |
|------|------|
```

### 4. Copilot adapter (optional)
If `.github/copilot-instructions.md` exists (Copilot detected), ALSO copy to `.github/instructions/conventions.instructions.md` with `applyTo: "**/*"` frontmatter.

### 5. Persist
- Write `openspec/conventions.md` (canonical).
- Optionally write `.github/instructions/conventions.instructions.md` (Copilot adapter).
- Both files are version-controlled.

## Rules
- Compact rules: 5–15 lines per skill — concise, actionable, no filler
- If no skills found: write empty registry with placeholder message
- Do NOT include sdd-* skills in the user-visible skills table (they are orchestrator-managed)
